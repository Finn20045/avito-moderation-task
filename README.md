# Avito Moderation Dashboard

Система модерации объявлений, разработанная в рамках тестового задания.
Приложение позволяет просматривать список объявлений, фильтровать их, принимать решения по модерации и отслеживать статистику эффективности.

## Стек технологий

- **Frontend:** React 18, TypeScript, Vite
- **UI Kit:** Material UI (MUI v6)
- **State Management & API:** TanStack Query (React Query)
- **Routing:** React Router v6
- **Charts:** Recharts
- **Forms:** React Hook Form
- **Animation:** Framer Motion
- **Containerization:** Docker & Docker Compose

## Обоснование выбора технологий

В проекте использован современный стек для обеспечения производительности, типобезопасности и удобства разработки.

- **Vite:** Выбран вместо CRA/Webpack из-за значительно более высокой скорости сборки и HMR (Hot Module Replacement), что критично для DX.
- **TypeScript:** Использован для статической типизации. Это позволяет избегать runtime-ошибок, упрощает рефакторинг и служит самодокументацией кода (особенно важно для `FetchAdsParams` и API ответов).
- **TanStack Query (React Query):** Выбран для управления серверным состоянием. В отличие от Redux/Effect, он "из коробки" предоставляет кэширование, дедупликацию запросов, состояния загрузки/ошибок и поллинг (для Real-time фич), что значительно сократило объем boilerplate-кода.
- **Material UI (MUI v6):** Использование готовой дизайн-системы позволило сосредоточиться на бизнес-логике, а не на верстке базовых компонентов (кнопок, инпутов, сеток). V6 выбран как наиболее актуальная версия.
- **React Hook Form:** Выбран для работы с формами (в модальном окне) из-за производительности (меньше ререндеров) и удобной валидации.
- **Recharts:** Библиотека выбрана за декларативный подход (компоненты React), который отлично интегрируется в JSX и легко настраивается.
- **Framer Motion:** Простой и декларативный способ добавить анимации (появление карточек, переходы страниц) без сложного CSS.

## Функциональные возможности

### Основное
- **Список объявлений:** Пагинация, поиск по названию, фильтрация (по категории, статусу, диапазону цен), сортировка.
- **Детальный просмотр:** Галерея изображений, таблица характеристик, информация о продавце.
- **Модерация:**
  - Одобрение (`Approve`)
  - Отклонение с обязательным указанием причины (`Reject`)
  - Отправка на доработку с комментарием (`Request Changes`)
  - Просмотр истории модерации.

### Дополнительно
- **Статистика:** Графики активности, распределения решений и категорий.
- **Экспорт данных:** Скачивание отчетов в форматах CSV и PDF.
- **Тёмная тема:** Переключатель режимов с сохранением настроек в LocalStorage.
- **Горячие клавиши:**
  - `A` — Одобрить
  - `D` — Отклонить
  - `←` / `→` — Навигация между объявлениями
  - `/` — Фокус на поле поиска
- **Real-time:** Автоматическое обновление статусов и уведомление о появлении новых объявлений (Short Polling).
- **Docker:** Полная контейнеризация клиента и сервера.

## Особенности реализации

- **Архитектура:** Проект структурирован по методологии Feature-Sliced Design (упрощенная версия) для лучшей масштабируемости и читаемости.
- **URL-синхронизация:** Состояние фильтров хранится в URL, что позволяет делиться ссылками на конкретную выборку объявлений и использовать навигацию браузера.
- **Optimistic Updates:** Использование React Query для управления серверным состоянием и автоматического обновления данных после мутаций.
- **Производительность:** Для массовых операций (Bulk Actions) используется параллельная отправка запросов (`Promise.all`).

## Запуск проекта

### Вариант 1: Через Docker (Рекомендуется)
Самый быстрый способ, не требует установки Node.js.

```bash
docker-compose up --build
```
Приложение будет доступно по адресу: http://localhost:8080

### Вариант 2: Локальный запуск (Dev mode)

Для запуска потребуется два терминала.

1. **Запуск сервера:**

```bash
cd tech-int3-server
npm install
npm start
```
Сервер запустится на порту 3001.

2. **Запуск клиента:**

Откройте новый терминал в корневой папке проекта:
```bash
npm install
npm run dev
```
Клиент запустится на http://localhost:5173
